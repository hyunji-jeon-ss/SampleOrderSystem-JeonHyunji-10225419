# 커밋 메시지 컨벤션

커밋 메시지는 `<헤더> 내용` 형식으로 작성합니다. 헤더는 꺾쇠괄호(`<>`)로 감싸고, 그 뒤에 한 줄로 변경 내용을 작성합니다.

```
<헤더> 변경 내용 요약
```

## 헤더 종류
아래 헤더 종류 외에는 절대 사용하지 않는다.

| 헤더 | 설명 |
| --- | --- |
| `<FEATURE>` | 새로운 기능 추가 |
| `<FIX>` | 버그 수정 |
| `<DOCS>` | 문서 수정 |
| `<STYLE>` | 코드 포맷팅, 세미콜론 등 스타일 변경 (기능 변경 없음) |
| `<REFACTOR>` | 코드 리팩토링 |
| `<TEST>` | 테스트 코드 추가/수정 |
| `<CHORE>` | 빌드, 설정 등 기타 변경 |

## 예시

```
<FIX> 긴급 Bug Fix
<FEAT> 로그인 기능 추가
<DOCS> README에 사용법 추가
```
